# Hyper-Weak-datasets

This repository hosts the documentation and GitHub Pages website for **Hyper-Weak-datasets**, a collection of weak-supervision hyperspectral image datasets generated from fully supervised hyperspectral benchmarks.

## Overview

**Hyper-Weak-datasets** is designed for research on weakly supervised hyperspectral image classification. The dataset generation pipeline converts clean pixel-level ground-truth labels into multiple weak-label settings, making it possible to evaluate models under different levels and types of supervision.

The dataset is generated by:

```text
D:\MasterProgram\Paper6\Python_Project_P6\data\process_all.py
```

The pipeline currently supports the following hyperspectral datasets:

- `AeroRIT`
- `Chikusei`
- `Houston2013`
- `Houston2018`
- `Pavia_Centre`
- `Pavia_University`
- `Washington_DC`

## Weak-Supervision Settings

The generation script supports three weak-supervision scenarios:

- **Incomplete labels:** sparse pixel-level labels sampled from the clean label map. The default sampling rates are `1%`, `5%`, `10%`, `20%`, and `50%`.
- **Inaccurate labels:** noisy labels generated from clean labels, including symmetric noise, asymmetric noise, boundary noise, and a noise-type mask.
- **Inexact labels:** coarse supervision generated as scribble-level labels and point-level labels.

These weak labels are generated together with the original image data and clean labels, allowing controlled comparison between full supervision and weak supervision.

## Dataset Structure

The generated dataset root contains one folder per hyperspectral dataset. Label arrays and image cubes are saved as GeoTIFF files (`.tif`):

```text
F:\Hyper_Weak\
+-- <dataset_name>\
    +-- images\
    |   +-- image.tif
    |   +-- reflectance.tif  (if have)
    |   +-- radiance.tif (if have)
    +-- clean_labels\
    |   +-- labels.tif
    |   +-- label_mapping.json
    +-- incomplete_labels\
    |   +-- incomplete_labels_1percent.tif
    |   +-- incomplete_labels_5percent.tif
    |   +-- incomplete_labels_10percent.tif
    |   +-- incomplete_labels_20percent.tif
    |   +-- incomplete_labels_50percent.tif
    +-- inaccurate_labels\
    |   +-- inaccurate_labels.tif
    |   +-- symmetric_noise_labels.tif
    |   +-- asymmetric_noise_labels.tif
    |   +-- boundary_noise_labels.tif
    |   +-- noise_type_mask.tif
    +-- inexact_labels\
    |   +-- scribble_labels.tif
    |   +-- point_labels.tif
    +-- incomplete_labels_metadata.json
    +-- inaccurate_labels_metadata.json
    +-- inexact_labels_metadata.json
```

Depending on the source dataset, `images\` may contain `image.tif`, `reflectance.tif`, `radiance.tif`, or a subset of these files.

### Image and Clean-Label Files

- `images\image.tif`: the hyperspectral image cube when the source loader provides a single image array.
- `images\reflectance.tif`: the reflectance hyperspectral cube when reflectance data are provided.
- `images\radiance.tif`: the radiance hyperspectral cube when radiance data are provided.
- `clean_labels\labels.tif`: the original dense pixel-level ground-truth label map.
- `clean_labels\label_mapping.json`: the mapping between numeric class IDs and semantic class names, when provided by the source dataset loader.

### Incomplete-Label Files

Incomplete labels simulate sparse pixel-level supervision. Unselected pixels are set to the background value `0`, while selected pixels keep their original class labels.

- `incomplete_labels_1percent.tif`: keeps approximately 1% of labeled pixels per class; this is the most sparse incomplete-label setting.
- `incomplete_labels_5percent.tif`: keeps approximately 5% of labeled pixels per class.
- `incomplete_labels_10percent.tif`: keeps approximately 10% of labeled pixels per class.
- `incomplete_labels_20percent.tif`: keeps approximately 20% of labeled pixels per class.
- `incomplete_labels_50percent.tif`: keeps approximately 50% of labeled pixels per class; this is the densest incomplete-label setting.

The incomplete labels are generated with nested stratified sampling, so lower sampling-rate labels are subsets of higher sampling-rate labels when the same random seed is used.

### Inaccurate-Label Files

Inaccurate labels simulate label noise. The default configuration uses a total noise ratio of `30%` over valid labeled pixels and combines three noise types.

- `inaccurate_labels.tif`: the combined noisy label map containing symmetric, asymmetric, and boundary noise.
- `symmetric_noise_labels.tif`: labels corrupted by random class replacement. This file simulates random noise such as sensor-related uncertainty or random label corruption, where a pixel may be assigned to any incorrect class without using spectral or spatial context.
- `asymmetric_noise_labels.tif`: labels corrupted toward spectrally similar classes. This simulates class confusion caused by similar spectral signatures between land-cover categories.
- `boundary_noise_labels.tif`: labels corrupted near class boundaries. This simulates boundary ambiguity, registration error, mixed pixels, or imprecise manual delineation around object edges.
- `noise_type_mask.tif`: a mask recording which noise type was applied to each pixel. The encoding is `0 = clean`, `1 = symmetric`, `2 = asymmetric`, and `3 = boundary`.

### Inexact-Label Files

Inexact labels simulate coarse annotations that identify only partial object or region information instead of dense pixel-level labels.

- `scribble_labels.tif`: scribble-level labels generated from the skeleton of each class region. These labels simulate annotation traces where only thin strokes inside class regions are labeled.
- `point_labels.tif`: point-level labels generated by selecting one representative point from each connected component. These labels simulate minimal supervision where each object or region is annotated by a single labeled point.

### Metadata Files

- `incomplete_labels_metadata.json`: records the incomplete-label generation parameters, including sampling rates, background value, seed, and output label format.
- `inaccurate_labels_metadata.json`: records the inaccurate-label generation parameters, including noise ratio, noise weights, boundary-shift settings, background value, seed, and output label format.
- `inexact_labels_metadata.json`: records the inexact-label generation parameters, including background value and output label format.

## Dataset Description

- **Dataset name:** Hyper-Weak-datasets
- **Version:** v1.0
- **Domain:** hyperspectral remote sensing
- **Task:** weakly supervised hyperspectral image classification
- **Input data:** hyperspectral image cubes
- **Label types:** clean labels, incomplete labels, inaccurate labels, and inexact labels
- **Generation script:** `process_all.py`

## Generation

The dataset can be generated from the processing script:

```bash
python D:\MasterProgram\Paper6\Python_Project_P6\data\process_all.py
```

By default, the script can be configured through `DATASET_LOADERS` and the corresponding weak-label configuration dictionaries:

- `DEFAULT_INCOMPLETE_CONFIG`
- `DEFAULT_INACCURATE_CONFIG`
- `DEFAULT_INEXACT_CONFIG`

The default incomplete-label configuration uses a fixed random seed of `42` and sampling rates of `0.01`, `0.05`, `0.10`, `0.20`, and `0.50`.
