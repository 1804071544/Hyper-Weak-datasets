# Hyper-Weak-datasets

This repository hosts the documentation and GitHub Pages website for **Hyper-Weak-datasets**, a collection of weak-supervision hyperspectral image datasets generated from fully supervised hyperspectral benchmarks.

## Overview

**Hyper-Weak-datasets** is designed for research on weakly supervised hyperspectral image classification. The dataset generation pipeline converts clean pixel-level ground-truth labels into multiple weak-label settings, making it possible to evaluate models under different levels and types of supervision.

The dataset is generated by:

```text
D:\MasterProgram\Paper6\Python_Project_P6\data\process_all.py
```

The pipeline currently supports the following hyperspectral datasets:

- `AeroRIT`
- `Chikusei`
- `Houston2013`
- `Houston2018`
- `Pavia_Centre`
- `Pavia_University`
- `Washington_DC`

## Weak-Supervision Settings

The generation script supports three weak-supervision scenarios:

- **Incomplete labels:** sparse pixel-level labels sampled from the clean label map. The default sampling rates are `1%`, `5%`, `10%`, `20%`, and `50%`.
- **Inaccurate labels:** noisy labels generated from clean labels, including symmetric noise, asymmetric noise, boundary noise, and a noise-type mask.
- **Inexact labels:** coarse supervision generated as scribble-level labels and point-level labels.

These weak labels are generated together with the original image data and clean labels, allowing controlled comparison between full supervision and weak supervision.

## Dataset Structure

The generated dataset root contains one folder per hyperspectral dataset:

```text
F:\Hyper_Weak\
+-- <dataset_name>\
    +-- images\
    +-- clean_labels\
    +-- incomplete_labels\
    +-- inaccurate_labels\
    |   +-- inaccurate_labels\
    |   +-- symmetric_noise_labels\
    |   +-- asymmetric_noise_labels\
    |   +-- boundary_noise_labels\
    |   +-- noise_type_mask\
    +-- inexact_labels\
```

## Dataset Description

- **Dataset name:** Hyper-Weak-datasets
- **Version:** v1.0
- **Domain:** hyperspectral remote sensing
- **Task:** weakly supervised hyperspectral image classification
- **Input data:** hyperspectral image cubes
- **Label types:** clean labels, incomplete labels, inaccurate labels, and inexact labels
- **Generation script:** `process_all.py`

## Generation

The dataset can be generated from the processing script:

```bash
python D:\MasterProgram\Paper6\Python_Project_P6\data\process_all.py
```

By default, the script can be configured through `DATASET_LOADERS` and the corresponding weak-label configuration dictionaries:

- `DEFAULT_INCOMPLETE_CONFIG`
- `DEFAULT_INACCURATE_CONFIG`
- `DEFAULT_INEXACT_CONFIG`

The default incomplete-label configuration uses a fixed random seed of `42` and sampling rates of `0.01`, `0.05`, `0.10`, `0.20`, and `0.50`.
